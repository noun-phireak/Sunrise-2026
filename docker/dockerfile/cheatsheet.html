🐳 Dockerfile Instructions

A one-page reference for every instruction — syntax, purpose, and the gotcha that matters.

Foundation & filesystem
01FROM

Sets the base image. Every Dockerfile starts here.

FROM image:tag [AS name]

Always pin a tag — never rely on the moving :latest.

02WORKDIR

Sets the working directory for later instructions.

WORKDIR /app

Creates the dir. Never use RUN cd — the change is lost.

03COPY

Copies files from the build context into the image.

COPY src dest/

Src must be inside the context — ../file fails. Prefer over ADD.

04ADD

Like COPY, but auto-extracts tars & fetches URLs.

ADD stuff.tar.gz /dest/

Only reach for it to unpack a local archive; otherwise use COPY.

05RUN

Executes a command at build time — creates a layer.

RUN apk add --no-cache curl

Chain with && and clean up in the same RUN to keep layers small.

Configuration
06ENV

Sets env vars that persist into the running container.

ENV KEY=value

Override with -e at run. Never store secrets here.

07ARG

Build-time variable. Does not persist to run time.

ARG NAME=default

Pass --build-arg. Assign to an ENV to keep it at run time.

08EXPOSE

Documents the port(s) the container listens on.

EXPOSE 80/tcp

Doesn't publish — you still need -p 8080:80 at run.

13LABEL

Adds metadata (key=value) to the image.

LABEL version="1.0"

Use OCI keys (org.opencontainers.image.*); read via docker inspect.

Startup behavior
09CMD

Default command/args when the container runs.

CMD ["echo", "hi"]

Overridden by run args. Only the last CMD counts.

10ENTRYPOINT

The executable that always runs; CMD becomes its args.

ENTRYPOINT ["echo"] CMD ["world"] # default arg

Replace it at run with --entrypoint. Prefer exec form.

16STOPSIGNAL

The signal sent to stop the container.

STOPSIGNAL SIGTERM

Exec-form ENTRYPOINT makes your app PID 1 so it receives the signal.

14HEALTHCHECK

Tells Docker how to test the container is healthy.

HEALTHCHECK --interval=10s \ CMD wget -q -O- localhost || exit 1

Exit 0 = healthy, 1 = unhealthy. Shows in docker ps.

Runtime environment & security
11VOLUME

Declares a mount point for persistent/shared data.

VOLUME ["/data"]

Data survives container removal. Write files before declaring it.

12USER

Sets the user that RUN/CMD/ENTRYPOINT run as.

USER appuser

Don't run as root. Create the user first, then switch.

17SHELL

Changes the default shell for shell-form commands.

SHELL ["/bin/bash", "-euo", "pipefail", "-c"]

Only affects shell form — exec form never uses a shell.

Advanced & legacy
19COPY --frommulti-stage

Copies artifacts between build stages.

FROM golang AS build ... COPY --from=build /app/bin .

Build fat, ship slim — final image drops the toolchain & source.

15ONBUILD

Registers a trigger that fires in a child image.

ONBUILD COPY . /app

Runs only in direct children, right after their FROM.

18MAINTAINERdeprecated

Old way to set the image author.

# use this instead: LABEL maintainer="you@example.com"

Deprecated since 2017 — migrate to LABEL maintainer.
